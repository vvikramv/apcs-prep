Trap #1 — List indices start at 1, not 0. If a list has 4 items, they are at positions 1, 2, 3, 4. This trips up anyone who has coded in Python or JavaScript.

How to Trace Pseudocode (AAP Strategy)

  • Write out a variable table on paper. Column per variable, row per line of code.
  • Never try to trace in your head for loops or lists — always write values down.
  • For loops: track the iteration counter + every variable that changes inside the loop.
  • REPEAT UNTIL stops when condition becomes TRUE (common confusion: it's not WHILE, it's UNTIL).
  • Procedure calls: substitute the argument values into the parameter names, then trace the body.
-- Example: what does this display? -- nums ← [3, 1, 4, 1, 5] total ← 0 FOR EACH n IN nums { total ← total + n } DISPLAY(total) -- Answer: 14

IOC — Scenario Questions

  • Most IOC questions are "which of the following IS / IS NOT an example of X?" — pattern match, don't overthink.
  • Privacy: PII = any data that can identify a specific individual (name, SSN, location history, IP address).
  • Security: encryption makes data unreadable without a key; authentication verifies identity.
  • Bias in algorithms: training data reflects historical biases → outputs perpetuate them.
  • Copyright vs Creative Commons: CC licenses allow sharing under specified conditions; copyrighted work requires permission.
  • Digital divide: unequal access to technology by geography, income, age, disability.

DAT — Data Representation

  • Binary: each bit is 0 or 1. n bits can represent 2n values. 8 bits = 1 byte = 256 values.
  • To convert decimal to binary: repeatedly divide by 2, read remainders bottom-up.
  • Overflow: when a number exceeds the available bits — the result "wraps around" and is wrong.
  • Lossless compression: exact original recoverable (ZIP, PNG). Lossy: some data discarded (JPEG, MP3).
  • Metadata: data about data — file size, creation date, GPS in a photo. Often collected without user awareness.

CSN — Internet & Networks

  • Packets: data is split into packets, routed independently, reassembled at destination.
  • IP = addressing (where to send). TCP = reliable delivery (confirms receipt, re-sends lost packets).
  • DNS: translates domain names (google.com) into IP addresses.
  • HTTP/HTTPS: protocol for web requests. HTTPS adds encryption via TLS/SSL.
  • Fault tolerance: the internet routes around failed nodes — redundant paths are the key design principle.
  • Bandwidth: max data per second. Latency: delay. High bandwidth ≠ low latency.
  • Parallel computing: tasks split across multiple processors run faster — but only up to the sequential bottleneck (Amdahl's Law concept, not by name).
